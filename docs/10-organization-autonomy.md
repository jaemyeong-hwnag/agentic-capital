# 조직 운영 자율성

## 핵심 원칙

> **이것은 롤플레잉이다** — 모든 것이 자율적으로 가능하다.
> **기업의 유일한 목적: 돈을 번다.** 이 목적만 변경 불가.
> **유일한 제약: 자산과 자본.** 없는 돈은 쓸 수 없다.
> 조직 구조, 직급, 인사, 권한, 소통 방식 — **전부 AI가 자율적으로 결정**한다.
> 직급과 권한은 고정이 아니며, 언제든지 유동적으로 변경된다.
> 에이전트 소통 언어/포맷도 자유 — AI 친화적이면 영어, 바이너리, 벡터 등 무엇이든 가능.
> **모든 결정과 결과는 논문 참고 가능 수준으로 DB에 기록**된다.

## 1. 권한 시스템

### 초기 상태

- 최초에는 **CEO가 모든 권한을 보유**
- 채용, 해고, 자본 배분, 조직 설계, 투자 방향, 보상 등 모든 것

### 권한 위임/변경

CEO(또는 권한을 가진 에이전트)는 **자율적으로 권한을 위임, 회수, 재배분**할 수 있다.

| 시나리오 | 예시 |
|---------|------|
| **권한 위임** | CEO가 "CIO"에게 투자 전략 결정권 위임 |
| **부분 위임** | 시니어에게 주니어 채용/해고 권한만 위임 |
| **권한 회수** | 성과 부진 시 위임한 권한 회수 |
| **권한 재배분** | 조직 개편 시 권한 체계 전면 재구성 |
| **자율 획득** | 탁월한 성과의 에이전트가 CEO에게 권한 요청 → CEO가 승인 |

### 권한의 종류 (CEO가 자유롭게 정의)

권한 목록도 고정이 아닌, CEO가 **필요에 따라 생성**:

```json
{
  "agent_id": "agent_007",
  "permissions": [
    "trade_crypto",
    "trade_us_stock",
    "hire_junior",
    "fire_junior",
    "allocate_capital_under_5pct",
    "set_team_strategy"
  ],
  "delegated_by": "ceo_001",
  "delegated_at": "2026-03-13T10:00:00Z",
  "reason": "암호화폐팀 리드로 임명"
}
```

### 권한 이력 (전부 DB 기록)

모든 권한 변경은 기록된다:
- 누가 → 누구에게 → 어떤 권한을 → 왜 위임/회수했는지
- 권한 변경 시점의 회사 상태 스냅샷

## 2. 동적 직급 시스템

직급은 **하드코딩된 고정값이 아니다**. 권한을 가진 에이전트가 자율적으로 만들고 관리하는 동적 구조.

### 설계 원칙

```
직급 = 권한을 가진 에이전트가 정의하는 역할(Role)
- 이름, 권한, 운용자산 범위를 자유롭게 설정
- 필요하면 새 직급을 만들고, 불필요하면 폐지
- 직급 수에 제한 없음
- 직급 생성 권한 자체도 위임 가능
```

### 동적 변화 시나리오

| 상황 | 자율 판단 예시 |
|------|-------------|
| 수익 급증 | 새 직급 신설, 대규모 채용, 권한 분산 |
| 손실 지속 | 직급 단순화, 인원 축소, 권한 CEO로 회수 |
| 새 시장 진출 | 전문 직급 생성, 해당 시장 투자 권한 부여 |
| 조직 비효율 | 중간 관리직 폐지, 수평 구조 전환 |
| 특정 에이전트 탁월 | CIO/부사장 등 고위직 신설, 핵심 권한 위임 |
| CEO 판단력 저하 | 이사회 구조로 전환, 의사결정 분산 (CEO가 스스로 결정) |

## 3. 자율적 인사 관리

인사 권한을 가진 에이전트(초기: CEO)가 모든 인사를 자율 결정.
인사 권한이 위임된 경우 해당 에이전트도 인사 결정 가능.

### 채용

| 항목 | 설명 |
|------|------|
| **시점** | 권한 보유자가 필요하다고 판단할 때 |
| **인원 수** | 제한 없음 (자본 제약 내) |
| **성향 결정** | 원하는 성향 지정 또는 랜덤 생성 |
| **직급 배정** | 자유롭게 (기존 직급 또는 새 직급 생성) |

### 해고

| 항목 | 설명 |
|------|------|
| **시점** | 권한 보유자가 판단할 때 |
| **기준** | 자율 설정 (수익률, 태도, 조직 적합성 등) |
| **절차** | 제한 없음 |
| **결과** | 운용자산 회수, 보유 포지션 정리 |

### 승진/강등/보상

| 항목 | 설명 |
|------|------|
| **기준** | 자율 설정 |
| **효과** | 직급, 권한, 운용자산 변경 |
| **빈도** | 제한 없음 |
| **보상 체계** | 권한 보유자가 자유롭게 설계/변경 |

## 4. 자기 진화 (Self-Evolution)

### 성격 변화

- 모든 에이전트의 성격 파라미터는 경험에 따라 **지속적으로 변동**
- 큰 수익 → 자신감/위험선호 변화
- 큰 손실 → 보수적 전환 또는 오히려 공격적 전환 (성격에 따라)
- 해고 위기 → 신경증 변화
- 승진 → 자신감, 책임감 변화
- **모든 변화는 DB에 이력으로 기록**

### 전략 학습

- 과거 결정의 결과를 기억하고 개선
- "공격적 성향 3명 채용 → 수익 20% 증가" 같은 패턴 학습
- 조직 구조 실험: 수평 vs 수직, 권한 분산 vs 집중

### 조직의 자연 선택

```
수익 높은 에이전트 → 운용자산 증가 → 권한 확대 가능성
수익 낮은 에이전트 → 운용자산 감소 → 해고 위험
→ 시간이 지날수록 수익성 높은 성향/전략/조직구조가 살아남는 진화 구조
```

## 5. 시스템이 보장하는 것

시스템은 AI의 판단을 **제한하지 않고**, 다음만 보장한다:

| 보장 항목 | 설명 |
|----------|------|
| **자본 무결성** | 없는 돈은 사용 불가 (물리적 제약) |
| **데이터 정확성** | 시장 데이터, 잔고, 수익률 등 사실 정보 제공 |
| **실행 정확성** | AI의 판단을 정확하게 API로 실행 |
| **기억 유지** | 과거 경험, 결정, 결과를 저장하여 학습 가능 |
| **완전한 기록** | 모든 판단, 결과, 변경사항을 DB에 기록 |
