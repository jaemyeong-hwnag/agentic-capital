# 실제 투자 API 연동 가이드

## 핵심 원칙

> AI의 투자 판단에 대한 시스템 제약은 **자본(잔고)** 뿐이다.
> 시스템은 AI의 결정을 정확하게 실행하는 역할만 한다.

**파이프라인**: `[데이터 수집] → [AI 판단] → [자본 검증] → [API 실행]`

- 자본 검증: 잔고 초과 여부만 확인 (물리적 제약)
- 그 외 투자 비율, 횟수, 종목 등의 제한은 없음

---

## 1단계: 타겟 시장 선택

AI가 자율적으로 시장을 선택할 수 있도록 **모든 시장 API를 연결**한다.

| 시장 | API | 특징 |
|------|-----|------|
| 암호화폐 | Upbit, Binance | 24시간 운영, API 문서 우수, Python 라이브러리 완비 |
| 미국 주식 | Alpaca API | AI 트레이더 특화, 모의투자(Paper Trading) 완벽 지원 |
| 국내 주식 | 한국투자증권 Open API | REST API 제공, 웹 환경 친화적 |

에이전트가 어떤 시장에서 거래할지는 **에이전트의 자율 판단**.

---

## 2단계: 파이프라인 아키텍처 (3개 모듈)

### 모듈 1: Data Fetcher (현실 데이터 수집기)

- 증권사 API 호출
- 현재 계좌 잔고 조회
- 보유 종목 수익률 조회
- 타겟 종목 현재 가격 및 최근 뉴스 수집

### 모듈 2: Agent Brain (AI 판단)

- 수집한 현실 데이터를 텍스트로 가공
- DB에서 AI 성향 컨텍스트 + 과거 경험 로드
- AI가 자율적으로 판단 (매수/매도/관망, 종목, 금액 등)

### 모듈 3: Execution (실행)

- AI 응답 파싱
- **자본 검증만 수행** (잔고 >= 주문금액)
- 통과 시 API로 주문 실행

---

## 3단계: 시스템 보장 사항

시스템은 AI를 제한하지 않고, 다음만 보장한다:

| 항목 | 설명 |
|------|------|
| **자본 무결성** | 없는 돈은 사용 불가 (물리적 제약) |
| **데이터 정확성** | 시장 데이터, 잔고, 수익률 등 사실 정보 제공 |
| **실행 정확성** | AI의 판단을 정확하게 API로 실행 |
| **기억 유지** | 과거 경험, 결정, 결과를 저장하여 학습 가능 |
| **모의투자 환경** | 초기에는 Paper Trading으로 시스템 검증 |
