# 실제 투자 API 연동 가이드

## 아키텍처: Core vs Adapter

```
┌─────────────────────────────────────────────┐
│          CORE (메인 — 변경 불가)              │
│  AI 롤플레잉 시뮬레이션                       │
│  에이전트 성격/감정/의사결정/조직 자율운영       │
│  목적: 돈 버는 것                             │
└──────────────────┬──────────────────────────┘
                   │ Port (인터페이스)
┌──────────────────┴──────────────────────────┐
│        ADAPTERS (플러그인 — 교체 가능)         │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐       │
│  │ KIS     │ │ Binance │ │ Alpaca  │  ...  │
│  │ Adapter │ │ Adapter │ │ Adapter │       │
│  └─────────┘ └─────────┘ └─────────┘       │
│  증권사/거래소 API는 언제든 교체 가능           │
└─────────────────────────────────────────────┘
```

> **메인**: AI 롤플레잉으로 돈 버는 것
> **트레이딩 시스템(증권사 API)**: 언제든지 변경 가능한 외부 어댑터

---

## 핵심 원칙

> AI의 투자 판단에 대한 시스템 제약은 **자본(잔고)** 뿐이다.
> 투자 종목, 방법, 자료 조사 방법 등 **모든 것을 가리지 않는다**.
> 시스템은 AI의 결정을 정확하게 실행하는 역할만 한다.

**파이프라인**: `[데이터 수집] → [AI 판단] → [자본 검증] → [API 실행]`

- 자본 검증: 잔고 초과 여부만 확인 (물리적 제약)
- 투자 종목, 비율, 횟수, 시점, 방법, 조사 방법 등 **일체 제한 없음**

---

## 타겟 시장

AI가 자율적으로 시장을 선택할 수 있도록 **모든 시장 API를 연결**한다.

| 시장 | API | Adapter | 특징 |
|------|-----|---------|------|
| 암호화폐 | Upbit, Binance | `ccxt` | 24시간 운영, 글로벌 접근 |
| 미국 주식 | Alpaca API | `alpaca-py` | Paper Trading 지원, AI 친화 |
| 국내 주식 | 한국투자증권 Open API | `python-kis` | REST+WebSocket, async 네이티브 |

> 에이전트가 어떤 시장에서 거래할지는 **에이전트의 자율 판단**.
> 증권사/거래소 API는 **언제든 다른 것으로 교체 가능** — Port만 구현하면 됨.

상세: [18 - 한국 증권사 API](18-korean-broker-api.md)

## 파이프라인 아키텍처 (3개 모듈)

### 모듈 1: Data Fetcher (현실 데이터 수집기)

- 증권사 API 호출 (Adapter를 통해)
- 현재 계좌 잔고 조회
- 보유 종목 수익률 조회
- 타겟 종목 현재 가격 및 최근 뉴스 수집

### 모듈 2: Agent Brain (AI 판단)

- 수집한 현실 데이터를 AI 친화적 포맷으로 가공
- DB에서 AI 성향 컨텍스트 + 과거 경험 로드
- AI가 자율적으로 판단 (매수/매도/관망, 종목, 금액 등)
- **조사 방법도 자율** — 기술적 분석, 펀더멘탈, 감성, 뉴스 등 가리지 않음

### 모듈 3: Execution (실행)

- AI 응답 파싱
- **자본 검증만 수행** (잔고 >= 주문금액)
- 통과 시 Adapter를 통해 주문 실행

## 시스템 보장 사항

시스템은 AI를 제한하지 않고, 다음만 보장한다:

| 항목 | 설명 |
|------|------|
| **자본 무결성** | 없는 돈은 사용 불가 (물리적 제약) |
| **데이터 정확성** | 시장 데이터, 잔고, 수익률 등 사실 정보 제공 |
| **실행 정확성** | AI의 판단을 정확하게 Adapter → API로 실행 |
| **기억 유지** | 과거 경험, 결정, 결과를 저장하여 학습 가능 |
| **모의투자 환경** | 초기에는 Paper Trading으로 시스템 검증 |
| **어댑터 교체** | 증권사 API 변경 시 Core 변경 없이 Adapter만 교체 |
