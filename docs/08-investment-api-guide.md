# 실제 투자 API 연동 가이드

## 핵심 원칙

> LLM이 직접 API에 접근하는 것은 **매우 위험**. AI 환각(Hallucination)으로 인해 예상치 못한 주문이 실행될 수 있음.

**필수 파이프라인**: `[데이터 수집] → [AI 판단] → [시스템 검증] → [API 실행]`

---

## 1단계: 타겟 시장 선택

| 우선순위 | 시장 | API | 특징 |
|---------|------|-----|------|
| 1 (추천) | 암호화폐 | Upbit, Binance | 24시간 운영, API 문서 우수, Python 라이브러리 완비 |
| 2 | 미국 주식 | Alpaca API | AI 트레이더 특화, 모의투자(Paper Trading) 완벽 지원 |
| 3 | 국내 주식 | 한국투자증권 Open API | REST API 제공, 웹 환경 친화적 |

---

## 2단계: 파이프라인 아키텍처 (3개 모듈)

### 모듈 1: Data Fetcher (현실 데이터 수집기)

- 증권사 API 호출
- 현재 계좌 잔고 조회
- 보유 종목 수익률 조회
- 타겟 종목 현재 가격 및 최근 뉴스 수집

### 모듈 2: Agent Brain (AI 판단 컨텍스트 생성기)

- 수집한 현실 데이터를 텍스트로 가공
- DB에서 AI 성향 컨텍스트 로드
- 프롬프트 조합 후 LLM에 판단 요청

**프롬프트 예시**:
> "너는 손실을 극도로 싫어하는 성향의 투자자다. 현재 잔고는 100만 원이고, 삼성전자가 오늘 -2% 하락 중이다. 관련된 뉴스는 다음과 같다. [뉴스 데이터]. 매수/매도/관망 중 하나를 선택하고 이유를 JSON 형태로 출력해라."

### 모듈 3: Execution & Guardrail (실행 및 안전장치)

- AI 응답 JSON 파싱
- **안전 검사 로직 통과 시에만** 실제 주문 실행

---

## 3단계: 안전장치 (Guardrails) - 필수

| 안전장치 | 설명 |
|---------|------|
| **1회 투자 한도** | 성향별 차등 (보수적 5% ~ 공격적 40%), 절대 상한 50% |
| **최소 예비금** | 운용자산의 20%는 항상 현금 보유 필수 |
| **동시 포지션 제한** | 최대 보유 종목 수 = 위험선호도 기반 (1~5개) |
| **단일 종목 집중 한도** | 전체 자본의 30% 초과 불가 |
| **일일 최대 거래 횟수** | 에이전트당 최대 5회 |
| **최소 보유 시간** | 암호화폐 5분 / 주식 1일 |
| **레버리지 제한** | 위험선호 80+ 만 허용, 최대 2배 |
| **감정 상태 제한** | 스트레스 80+ 시 냉각 기간, 자신감 90+ 시 한도 하향 |
| **모의투자 선행** | 최소 1~2달 모의투자 계좌로 운영 후 실거래 전환 |
