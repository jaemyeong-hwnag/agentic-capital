테스트 커버리지를 분석하고 80% 이상을 달성하도록 테스트를 보강합니다.

## 실행 순서

1. 현재 커버리지 상태를 확인합니다:
   ```
   python -m pytest tests/ --cov=src --cov-report=term-missing --cov-report=html
   ```

2. 커버리지 80% 미만인 모듈을 우선순위별로 정리합니다:
   - 핵심 비즈니스 로직 (도메인 레이어) → 최우선
   - 서비스 레이어 → 높음
   - 인프라/어댑터 레이어 → 보통

3. 누락된 테스트를 작성합니다:
   - 각 public 메서드에 대한 happy path 테스트
   - 경계값 테스트 (boundary value)
   - 에러 케이스 테스트
   - 핵심 분기(branch) 커버리지 확보

4. 작성한 테스트를 실행하여 커버리지 변화를 확인합니다.

5. 전체 커버리지가 80% 이상이 될 때까지 반복합니다.

## 커버리지 제외 대상
- `__init__.py`
- 설정 파일 (config)
- 마이그레이션 파일
- 타입 정의 파일
