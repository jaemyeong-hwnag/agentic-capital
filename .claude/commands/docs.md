프로젝트 문서를 자동으로 정리하고 최신 상태로 동기화합니다.

## 실행 순서

1. **현재 문서 구조 파악**:
   - `docs/` 폴더의 모든 마크다운 파일 스캔
   - `readme.md` 내용 확인
   - 코드 내 docstring 및 주석 확인

2. **코드 변경사항 감지**:
   - `git diff` 또는 `git log`로 최근 변경된 파일 확인
   - 변경된 파일이 문서에 영향을 주는지 판단:
     - 새 모듈/클래스 추가 → API 문서 업데이트 필요
     - 설정 변경 → 설정 문서 업데이트 필요
     - 기술 스택 변경 → tech-stack 문서 업데이트 필요
     - DB 스키마 변경 → 데이터 모델 문서 업데이트 필요

3. **문서 업데이트**:
   - 변경사항에 맞게 관련 문서 수정
   - 새로운 주제가 있으면 적절한 문서 파일에 추가
   - 기존 문서와 코드가 불일치하면 코드 기준으로 수정

4. **문서 구조 검증**:
   - 깨진 링크 확인
   - 중복 내용 제거
   - 번호 체계 정합성 확인 (01-, 02-, ...)
   - 목차(readme.md)와 실제 문서 파일 동기화

5. **readme.md 업데이트**:
   - docs/ 내 문서 목록을 readme.md에 반영
   - 프로젝트 현재 상태 요약 업데이트

## 문서 파일 네이밍 규칙
- `XX-topic-name.md` 형식 (XX는 2자리 번호)
- 영문 소문자, 하이픈 구분
- 주제별로 하나의 파일 유지

## 문서 작성 규칙
- 한국어 작성
- 기술 용어는 영문 병기
- 테이블 적극 활용
- 코드 예시 포함
