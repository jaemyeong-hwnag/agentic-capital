---

모든 코드 변경 작업이 완료된 후 반드시 실행하는 후처리 파이프라인입니다.

## 실행 순서

### Phase 1: 테스트 실행

1. pytest로 전체 테스트를 실행합니다:
   ```
   python -m pytest tests/ -v --tb=short --cov=src --cov-report=term-missing --cov-fail-under=80
   ```

2. 실패한 테스트가 있으면:
   - 실패 원인을 분석합니다
   - 코드 버그이면 코드를 수정합니다
   - 테스트 버그이면 테스트를 수정합니다
   - 수정 후 다시 테스트를 실행합니다
   - 모든 테스트가 통과할 때까지 반복합니다

### Phase 2: 테스트 커버리지 확인

3. 전체 커버리지가 80% 이상인지 확인합니다.

4. 80% 미만이면:
   - 커버리지가 부족한 파일을 식별합니다
   - 누락된 테스트를 작성합니다:
     - public 메서드 happy path
     - 경계값 (boundary value)
     - 에러 케이스
     - 핵심 분기(branch) 커버리지
   - 작성한 테스트를 실행하여 커버리지 확인합니다
   - 80% 이상이 될 때까지 반복합니다

### Phase 3: 문서 정리

5. 코드 변경사항이 문서에 영향을 주는지 확인합니다:
   - `git diff --name-only` 로 변경된 파일 확인
   - 새 모듈/클래스 추가 → 관련 문서 업데이트
   - DB 스키마 변경 → `11-data-model.md` 업데이트
   - 기술 스택 변경 → `05-tech-stack.md` 업데이트
   - 설정 변경 → 관련 문서 업데이트

6. 영향받는 문서를 업데이트합니다:
   - 코드와 문서 불일치 시 코드 기준으로 수정
   - README.md의 문서 목록 동기화

### Phase 4: 커밋

7. 커밋 규칙에 따라 목적별로 분리하여 커밋합니다:
   - 코드 수정 → `fix:` 또는 `feat:` 커밋
   - 테스트 추가 → `test:` 커밋
   - 문서 변경 → `docs:` 커밋

8. 커밋 메시지: `<type>: <summary>` (명령형, 72자 이내)

### Phase 5: 결과 요약

9. 최종 결과를 요약합니다:
   - 테스트: 총 N개, 통과 N개, 실패 N개
   - 커버리지: N% (목표 80%)
   - 문서: 업데이트된 파일 목록
   - 커밋: 생성된 커밋 목록
