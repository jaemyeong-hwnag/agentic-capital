프로젝트의 테스트를 실행하고 커버리지를 확인합니다.

## 실행 순서

1. pytest를 사용하여 전체 테스트를 실행합니다:
   ```
   python -m pytest tests/ -v --tb=short --cov=src --cov-report=term-missing --cov-fail-under=80
   ```

2. 커버리지가 80% 미만인 파일을 식별합니다.

3. 커버리지가 부족한 파일에 대해:
   - 어떤 라인이 커버되지 않았는지 분석합니다
   - 누락된 테스트 케이스를 제안하거나 직접 작성합니다
   - edge case와 error path도 포함합니다

4. 테스트 실패가 있으면:
   - 실패 원인을 분석합니다
   - 코드 버그인지 테스트 버그인지 판단합니다
   - 수정 방안을 제시하거나 직접 수정합니다

5. 최종 결과를 요약합니다:
   - 총 테스트 수, 통과/실패 수
   - 전체 커버리지 퍼센트
   - 커버리지 80% 미달 파일 목록

## 주의사항
- 테스트는 반드시 독립적으로 실행 가능해야 합니다
- fixture와 mock은 최소한으로 사용합니다
- 통합 테스트는 실제 DB를 사용합니다 (mock 금지)
