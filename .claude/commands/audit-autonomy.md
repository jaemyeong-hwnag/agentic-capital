프로젝트 핵심 원칙 준수 여부를 전체 코드베이스에서 검사합니다.

## 핵심 원칙

> **유일한 목적: 돈 벌기. 유일한 제약: 자산/자본.**
>
> 모든 AI 에이전트는 최대 자율성을 가진다. 시스템이 AI의 행동을 차단하지 않는다.

## 검사 항목

### 1. 시스템 강제 제한 (CRITICAL)
src/ 전체에서 AI 행동을 시스템이 차단하는 코드를 검색:
- 시장 시간 기반 실행 차단 (`if not is_market_open(): return` 등)
- 감정/성격 상태 기반 거래 차단 (감정이 극단적이어도 시스템이 차단하지 않아야 함)
- 하드코딩된 거래 한도 (자본 기반이 아닌 것)
- 강제 행동 패턴 (`must hold X% cash`, `max trades per day` 등)

### 2. 허용되는 유일한 제약: 자산/자본
다음은 정상적인 제약이므로 위반이 아님:
- 잔고 부족으로 주문 거부/조정
- 보유하지 않은 종목 매도 거부
- API 레이트 리밋 (인프라 제약, AI 자율성 제한 아님)
- 외부 API 오류 처리

### 3. 정보 제공 vs 강제 (IMPORTANT)
시장 상태, 감정, 리스크 등은 AI에게 **정보로 전달**은 허용, **실행 차단**은 금지:
- `is_market_open()` → 로깅/프롬프트에 정보로 포함 ✅
- `if not is_market_open(): return` → 실행 차단 ❌
- 감정 상태를 LLM 프롬프트에 포함 ✅
- 감정 상태로 거래 함수 호출 차단 ❌

### 4. 하드코딩된 거래 대상 제한
- 특정 종목만 거래 가능하도록 제한하는 코드
- 특정 시장/자산군만 허용하는 코드
- 투자 전략을 시스템이 강제하는 코드

### 5. 권한/역할 기반 차단
- `has_permission()` 등으로 AI 행동을 차단하는 코드
- 역할에 따라 거래를 못하게 막는 코드
- (역할/권한 데이터 구조 자체는 OK — AI가 자율적으로 활용하면 됨)

## 검사 방법

1. `src/` 전체 Python 파일에서 `return`, `raise`, `block`, `restrict`, `limit`, `deny`, `reject`, `forbidden`, `not allowed` 등 차단 키워드 검색
2. 의사결정 파이프라인(`core/decision/`)에서 자본 외 제약 조건 확인
3. 시뮬레이션 엔진(`simulation/engine.py`)에서 실행 게이트 확인
4. 프롬프트(`core/decision/prompts.py`)에서 AI 자율성 제한 문구 확인

## 출력

- 발견된 위반을 심각도로 분류:
  - **CRITICAL**: 시스템이 AI 행동을 강제로 차단
  - **WARNING**: 자본 외 제약이 존재하나 현재 미사용
  - **INFO**: 잠재적 위반 가능성 (코드 변경 시 주의 필요)
- 각 위반에 대해 파일 경로, 라인 번호, 위반 내용, 수정 방안 제시
- CRITICAL 위반은 즉시 수정
