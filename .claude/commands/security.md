프로젝트의 보안 이슈를 종합 검토합니다.

## 검토 항목

### 1. 시크릿/크레덴셜 노출 확인
- git 히스토리에 API 키, 비밀번호, 토큰이 커밋된 적 있는지 확인:
  ```
  git log --all -p | grep -iE "(api_key|secret|password|token|sk-|AIza)" | head -50
  ```
- `.env` 파일들이 `.gitignore`에 포함되어 있는지 확인
- 코드에 하드코딩된 시크릿이 있는지 검색

### 2. .gitignore 검증
- 모든 `.env*` 파일이 무시되는지 확인
- 로그 파일(*.log), 데이터 덤프(*.sql, *.dump), 바이너리 파일이 무시되는지 확인
- IDE/에디터 설정 파일이 무시되는지 확인

### 3. 의존성 취약점
- `pip audit` 실행하여 알려진 취약점 확인
- 오래된 패키지 버전 확인

### 4. 코드 보안 패턴
- SQL Injection 가능성 (raw SQL 사용 여부)
- 입력 검증 누락 (사용자 입력이 직접 사용되는 곳)
- 에러 메시지에 민감한 정보 노출 여부
- API 키가 로그에 출력되는지 확인

### 5. 인프라 보안
- Docker Compose 설정의 포트 노출 확인
- 데이터베이스 비밀번호 강도 확인
- Redis 인증 설정 확인

### 6. API 키 관리
- 모든 API 키가 환경변수로 관리되는지 확인
- 키 로테이션 가능한 구조인지 확인

## 출력
- 발견된 이슈를 심각도(HIGH/MEDIUM/LOW)로 분류
- 각 이슈에 대한 수정 방안 제시
- 즉시 수정 가능한 이슈는 직접 수정
